Router-outlet Explained

The router-outlet directive in Angular acts as a placeholder where the router displays the component for the current route. It dynamically loads components based on the navigation. As you switch between routes, Angular transitions between components without a full page reload. For instance, with routes like `/home` and `/about` tied to `HomeComponent` and `AboutComponent` respectively, navigating between them swaps out the components where the router-outlet is. In many apps, you'll find static components (e.g., navigation bars or footers) that remain visible regardless of the route. They sit alongside the router-outlet in the template.

Example:
  • <app-navbar></app-navbar>
  • <router-outlet></router-outlet>
  • <app-footer></app-footer>

Lifecycle Hook Description
ngOnChanges() It is called every time the property input is set up or updated.
ngOnInit() It is called when we are "initializing" a component. It's the right moment to do a fetch [of the initial data].
ngAfterViewInit() Called once after the view (and child views) are rendered.
ngOnDestroy() Cleanup logic just before Angular destroys the directive or component. Unsubscribe from Observables and detach event handlers to avoid memory leaks. Called just before the instance is destroyed.
